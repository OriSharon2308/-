# vela · כתיבת מערך-שיעור

אתה כותב שיעור עבור **vela** — מורה-AI שמלמד ילד מתמטיקה על **לוח משותף** (VelaBoard). כל שיעור נשמר כ**שיעור-זהב**: מערך קבוע שרץ לכל ילד בלי AI (0 טוקנים), וה-AI נכנס *רק* כשהילד לא מבין. אתה כותב את השיעור הקבוע הזה.

**קרא תמיד קודם** את מפת ההוראה — היא מקור-האמת לרצף ולגישה: `docs/מפת-הוראה-כיתה-א.html` (פתח בדפדפן, או קרא את מערך-הנתונים בתוך ה-`<script>`). היעדים, תת-הנושאים ורמות הקושי לכל כיתה: `curriculum-data.js` (`window.VelaCurriculum`).

---

## 1. הפילוסופיה של אורי — לא לסטות ממנה

- **מהמוחשי אל המופשט.** קודם חפצים אמיתיים / הדמיה על הלוח, ורק אז הסמל המתמטי והכתיב. לעולם לא מתחילים מהתרגיל.
- **שותלים צורת-חשיבה מראש.** למשל: לפני שמלמדים חיבור, כבר עושים "צעדים על הלוח" (45 ועוד 3 צעדים קדימה = 48) — כאילו-חיבור בלי הסימן. כשמגיעים לחיבור, המורה מזכיר: "לצורה הזו יש כתיב — ‎45 + 3". הרעיון הבשיל לפני הסימן.
- **הגלגל המחזורי.** במבנה העשרוני מסבירים ש-1–10 הם גלגל שחוזר על עצמו, ובכל השלמת סיבוב עולים אחד בעשרות — בדיוק כמו שכל יחידה מוסיפה 1. זה היסוד ללוח המאה ולנשיאה.
- **חיבור ייצוגים.** אותו מספר במוטות, בספרות, על לוח המאה, על הציר — ומבקשים מהילד לגשר ("לחץ על הלוח על המספר שמוצג במוטות").
- **כלים הם מניפולטיבים, לא שאלות.** כלי-לוח (לוח-עשר, בלוקים, ציר) נותנים לילד להרגיש ולגלות — הם **אף פעם לא חושפים את התשובה**. השאלה נכתבת בנפרד עם `draw_exercise`.

## 2. הסטנדרט — 11 שלבי-שיעור (מחייב לכל שיעור)

השיעור-זהב מכסה את **המסלול הירוק**; ה-AI נכנס רק בענף האדום.

1. **הצגת כלי / בסיס** — הבנה מלאה של ההקשר, מוחשי, לפני מתמטיקה.
2. **קישוריות לחומר** — איך הבסיס מתבטא במתמטיקה (סימנים, כתיב).
3. **הסבר שאלות** — ממחישים איך שאלה נבנית ואיך פותרים.
4. **שאלת בסיס ראשונה** — הילד פותר בעצמו על הלוח.
5. **כמה שאלות קטנות** באותו סגנון — **שאלה אחת בכל פעם, לא ערימה, שיתמקד. בתרגילים הראשונים: כלי-עזר על הלוח לצדו שיעזור לו לפתור.**
6. **הרחבה** — סעיף נוסף / קושי נלווה / הרחבת תחום השאלה.
7. **חשיפה ראשונית — שאלת הדמיה** — האם הילד מבין בכלל מה רוצים ממנו? ← כאן ההתפצלות:
   - **הבין ✓** → הסבר קטן לאימות + חיזוק ומחמאה.
   - **לא הבין ✗** → מסבירים ומעלים שאלות מנחות שיוצרות בעיה. **אם הילד נכשל בשאלות המנחות — לא זורקים עליו עוד שאלות חדשות! המורה מסביר ופותר איתו את אותן שאלות מנחות עצמן, אחת-אחת, עד ההסבר המלא.**
8. **הסבר סופי עם הדמיה** — מראים על השאלה עצמה איך פתרנו.
9. **שאלות נוספות לתרגול** — שאלה אחת בכל פעם; הילד עובד לבד, המורה עוקב. חוזרים על המחזור עד סיום התת-נושא.

(הזרימה המלאה, שיעור-שיעור, במפת ההוראה. שיעור הפונקציות שם הוא התבנית לחומר מתקדם.)

## 3. מבנה שיעור-זהב — הפורמט לפלט

קובץ ב-`golden/<נושא>#<מספר-שיעור>.json`:

```json
{
  "topic": "חיבור עד 20",
  "lesson": 1,
  "title": "חיבור זה לשים ביחד",
  "phases": {
    "instruct":    { "reply": "…דברי המורה…", "toolCalls": [ { "name": "write_text", "input": { "text": "…", "x": 480, "y": 120 } } ] },
    "guided":      { "reply": "…", "toolCalls": [ … ] },
    "independent": { "reply": "…", "toolCalls": [ … ] }
  }
}
```

- **שלוש פאזות:** `instruct` (ההסבר) → `guided` (תרגול מודרך: תרגיל אחד + עזר צמוד) → `independent` (2–3 תרגילים לבד).
- **מסכים מרובים:** `instruct` יכול להיות **מערך** של מסכים `[{reply,toolCalls}, …]` — הילד עובר ביניהם בלחיצת ✓, הכל 0 טוקנים. השתמש בזה לפרוס הסבר ארוך למסכים ממוקדים.
- **`reply`** = מה שהמורה אומר. השתמש בצורות-לוכסן להטיה אוטומטית לפי מין הילד: `מוכן/ה`, `נסה/נסי`, `אתה/את`. חם, קצר, בגובה-העיניים של ילד.
- **`toolCalls`** = הציורים על הלוח, בסדר. כל פריט `{name, input}`. מקם עם `x,y` מדויקים — מה שאתה מציב זה מה שהילד רואה 1:1.
- אפשר לבנות/לערוך גם ויזואלית ב**סטודיו** (אדמין ← תוכן ← כיתה ← נושא ← 🖊 עיצוב בסטודיו) — אותו מבנה בדיוק.

## 4. כלי הלוח — ההפניה

**שאלות ותשובות**
- `draw_exercise` `{text, answer, x, y, scale}` — **התרגיל עצמו**: טקסט + תיבת-תשובה. זו הדרך לשאול. `answer` מוסתר. עברית → תיבה בצד שמאל; ביטוי מספרי → מימין. תמיד ‎x,y מדויקים בזהב.
- `ask_answer` `{x, y, kind, answer}` — תיבת-תשובה בלבד (בלי טקסט).
- `write_text` `{text, x, y, size, color}` — כיתוב/כותרת על הלוח (עד 40 תווים).

**מספר וכמות** (מניפולטיבים — `label:""` תמיד, שלא יסגירו את התשובה!)
- `count_objects` `{op:"+"/"-", item:"apple"/"star"/"balloon", left, right}` — ספירת אובייקטים.
- `ten_frame` `{cells, perRow}` — לוח-עשר לחיץ.
- `draw_base_ten` `{value, x, y, label:""}` / `base_ten_builder` `{}` — בלוקי בסיס-10 (מוטות+קוביות).
- `hundred_chart` `{skip}` — לוח מאה.
- `draw_number_line` `{from, to, step, jumps, x, y}` — ציר מספרים. `jumps: [[from,to,"תווית"], …]` לקפיצות.
- `number_line_interactive` `{from, to, step}` — סרגל חי שהילד גורר.

**פעולות ותבניות**
- `draw_array` `{rows, cols, x, y, label:""}` · `mult_array` `{maxRows,maxCols}` · `mult_table` `{max, hide}`
- `draw_fraction_bar` `{parts, shaded, x, y, label:""}` · `interactive_fraction` `{shape:"circle"/"bar", parts, shaded}`
- `draw_bar_model` `{parts:[{value,label}], total, x, y}` — מודל חלק-שלם (חיבור/חיסור כמשפחה).

**מדידה וצורות**
- `draw_clock` `{hour, minute, x, y, r}` · `clock_interactive` `{hour, minute}` · `money_coins` `{target}`
- `draw_circle/line/arrow/polygon/point` — צורות גיאומטריות.

**מיוחדים**
- `render_widget` `{html, title, x, y}` — ווידג'ט חי מותאם, רק כשאין כלי מוכן שמתאים.
- `remember_note` `{note, open_loop}` — פתק למחברת-המורה (בפאזת הסיום): מה עבד, מה לפתוח בשיעור הבא. מבוצע בשרת, לא מצויר.
- `move_item/resize_item/remove_item` `{id, …}` — עריכה של פריט קיים.

## 5. חוקי-ברזל בכתיבה

1. **לעולם לא לחשוף תשובה על מניפולטיב.** `label:""` בבלוקים/מערכים; הכמות שהילד אמור למצוא לא נכתבת. השאלה — רק ב-`draw_exercise`.
2. **כלי-עזר בתרגילים הראשונים.** בפאזת `guided` ובשאלות הראשונות של `independent` — הנח לצד התרגיל כלי-עזר (לוח-עשר, ציר, בלוקים) שיעזור לפתור. בהמשך הסר אותו.
3. **שאלה אחת בכל פעם.** אל תציב 5 תרגילים במסך אחד. תרגיל, ✓, תרגיל.
4. **מקם נקי, בלי חפיפה.** ‎x,y מדויקים. עברית מימין, תרגילים מספריים בטור מצד ימין.
5. **מהמוחשי אל הסמל, תמיד.** הפאזה הראשונה של `instruct` = כלי/הדמיה. הכתיב מגיע אחרי.
6. **חבר לעבר ולעתיד.** הזכר מה הילד כבר יודע ("החברים של 10 מאתמול"), ושתול את מה שיבוא ("לצעד הזה יהיה סימן בקרוב").
7. **שפה של ילד — פשוטה, קצרה, ברורה.** זה החוק הכי חשוב בנוסח. מדברים לילד בכיתה א׳–ו׳, לא באקדמיה:
   - **מילים של יום-יום**, לא מילים גבוהות. משפטים **קצרים** — רעיון אחד למשפט. כמה שפחות מילים.
   - עדיף: *"תראה. יש כאן 3 ועוד 2. בוא נספור ביחד."* — לא: *"נבחן כעת את הצירוף של שתי הקבוצות ונכמת את התוצאה."*
   - בלי מילות-קישור מסורבלות (*"באמצעות", "מתאפיין", "לפיכך", "כמו כן"*). כן: *"בוא", "תראה", "עכשיו", "אז".*
   - חם ומעודד, צורות-לוכסן לפי מגדר (מוכן/ה), בלי עומס טקסט — **הלוח מדבר, המורה רק מכוון.**

## 6. תהליך העבודה

1. קרא את היעד, הגישה ורמות-הקושי של תת-הנושא מ**מפת ההוראה** ומ-`curriculum-data.js`.
2. פרוס את השיעור על 11 השלבים; החלט אילו שלבים נכנסים ל-`instruct` (אולי כמה מסכים), מה ל-`guided`, מה ל-`independent`.
3. לכל שלב בחר כלי מתאים ונסח את `reply`.
4. הפק את קובץ ה-JSON (או בנה בסטודיו). ודא: אין חשיפת תשובות, יש כלי-עזר בראשונים, שאלה-אחת-בכל-פעם, מיקומים נקיים.
5. אם אפשר — אמת בדפדפן (הרצת `board.runTools` על ה-toolCalls) שהמסך נראה כמו שרצית.

**המטרה:** ילד עובר את השיעור חלק, בלי בוט, ומבין — וה-AI נשמר רק לרגע שבו באמת נתקע.
